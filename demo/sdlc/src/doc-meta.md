# Document

- 文档是对用户功能、系统模块等对象的描述。
- 文档应采用 Markdown 文件格式。
- 文档应包含如下 4 个部分：
  - 标题：应使用词组简要概括文档描述的对象，如“# 指定区间提交合并分析功能”。
  - 对象：
    - 应采用二级标题“## 对象”。
    - 应几句话清晰说明本文档描述什么。
  - 描述：
    - 应采用二级标题“## 描述”。
    - 应包含如下 3 个部分（三级标题）：
      - 行为：应描述对用户或外部模块可见的系统行为。
      - 设计：应描述行为实现的原理、架构或方法。
      - 实现：应描述具体的实现方式。
    - 应采用编号引用文件或片段以支持描述，如“[5]”。
  - 引用文件列表：
    - 应采用二级标题“## 引用文件”。
    - 应编号列出文件相对于项目根目录的路径，保持编号顺序且在本文档中唯一。
    - 如果只涉及一个文件的片段，可标记出行号范围。
    - 可采用目录或`*`等模式匹配减少文件列表长度。
    - 应包含如下 2 个部分：
      - 对象实现直接对应的文件或片段：采用三级标题“### 对象实现”。
      - 描述涉及的其他参考文件或片段：采用三级标题“### 其他参考”。
